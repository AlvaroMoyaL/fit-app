# Repo Rules

## Context Updates
- Antes de resumir o "leer contexto" del proyecto, actualiza `PROJECT_CONTEXT.md` si hubo cambios relevantes en arquitectura, UX, datos, reglas de negocio o flujos principales.
- No responder con un contexto desactualizado si el estado real del código cambió durante la sesión.

## Commit Scope
- No incluir archivos de exportación o datos personales temporales en commits, salvo que se pida explícitamente.
